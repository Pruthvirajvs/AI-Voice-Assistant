# Screenshots and Demo Guide

A strong GitHub project should show what the application looks like and how it works. Use this guide to create assets for the README.

## Recommended Screenshots

1. Assistant start screen
2. Listening state after clicking Start
3. Successful command recognition
4. Example output for weather, calculator, or opening a website
5. Static showcase page from `index.html`

## Recommended Demo Video

Record a 30-60 second video showing:

1. Launching the assistant
2. Saying your name
3. Running `open youtube`
4. Running `weather`
5. Running `take screenshot`
6. Closing the assistant

## File Locations

Create this folder in the repository when assets are ready:

```text
assets/screenshots/
```

Suggested names:

```text
assistant-start.png
assistant-listening.png
assistant-command.png
showcase-page.png
demo-video.mp4
```

## README Placement

After screenshots are added, place them near the top of `README.md` under a `Screenshots` section.

Example:

```markdown
## Screenshots

![Assistant start screen](assets/screenshots/assistant-start.png)
![Showcase page](assets/screenshots/showcase-page.png)
```
