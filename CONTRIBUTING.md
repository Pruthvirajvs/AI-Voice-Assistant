# Contributing

Thanks for considering a contribution to AI Voice Assistant.

## Workflow

1. Create a new branch from `main`.
2. Keep changes focused and easy to review.
3. Update documentation when behavior changes.
4. Test the project locally before opening a pull request.

## Code Style

- Prefer clear names over clever shortcuts.
- Keep user-facing text concise and helpful.
- Do not commit API keys, tokens, generated secrets, or private data.

## Pull Requests

A good pull request includes:

- A short summary of the change
- Screenshots for UI updates
- Notes about testing performed
- Any follow-up work that should be tracked
